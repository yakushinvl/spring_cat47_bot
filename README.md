# Электронное бюро пропусков (команда «Вобла»)

Чат-бот для мессенджера MAX, разработанный в рамках студенческого хакатона «Весенний код»: трек «МАКС». Сервис автоматизирует процесс подачи и согласования заявок на гостевые пропуска.

## Ключевые особенности

**Уникальные фишки (скорее всего)**:

- **Бесшовный интерфейс**: Использование редактирования сообщений (`editMessage`) для создания плавных переходов между этапами формы без захламления истории чата.
- **Referral-система**: Генерация специальных ссылок с предустановленными параметрами визита (дата, время, зона).
- **Защита от дребезга (спама запросами)**: Встроенный механизм игнорирования повторных нажатий на кнопки (интервал 800мс).
- **YAML Конфигурация**: Удобная настройка организации через `config.yaml` (изменили с json после комментария на втором митапе).

**Полезные особенности бота**:
- **Гибкое управление ролями**: Система разделения прав (Инициатор, Администратор ИБ, Технический администратор).
- **Право на забвение**: Возможность полного удаления всех пользовательских данных (профиль, заявки, шаблоны) одним нажатием.
- **Безопасный аудит**: Выгрузка истории действий с автоматическим обезличиванием персональных данных гостя для технической поддержки.

## Технологический стек

- **Платформа и ЯП**: Node.js (JavaScript)
- **Библиотека API**: `@maxhub/max-bot-api` (официальня библиотека MAX)
- **База данных**: PostgreSQL
- **Визуал**: `canvas` (для генерации графиков статистики)
- **Конфиг**: `js-yaml`

## Требования

- Node.js 18.x или выше
- PostgreSQL 14.x или выше

## Настройка

**Настройте окружение**:

Создайте файл `.env` в корне проекта по образцу:
```env
BOT_TOKEN=ваш_токен_max
PGUSER=postgres
PGHOST=localhost
PGDATABASE=bot_db
PGPASSWORD=ваш_пароль
PGPORT=5432
DEBUG=true
```

**Сконфигурируйте организацию**:

Отредактируйте `config.yaml` для настройки рабочих часов, доступных зон и ID главного администратора.
```yaml
organization:
  working_hours:
    monday: { start: '08:30', end: '20:30' } # время работы организации по дням (понедельник)
    tuesday: { start: '08:30', end: '20:30' }
    wednesday: { start: '08:30', end: '20:30' }
    thursday: { start: '08:30', end: '20:30' }
    friday: { start: '08:30', end: '20:30' }
    saturday: { start: '08:30', end: '17:30' }
    sunday: null
  access_zones:
    - "Адрес 1" # зоны/корпуса доступа
    - "Зона 2"
    - "и т.д."
  min_days_before_visit: 0 # кол-во дней до дня визита
initial_tech_admin_id: 123456789 # id тех.админа
```

## Безопасность и соответствие кейсу

Проект разработан с учетом требований к обработке данных:
- **Обезличивание**: Технические администраторы получают отчеты, где ФИО гостей заменены на `(данные скрыты)`.
- **Идентификация**: Бот содержит явные дисклеймеры о разработчике (команда «Вобла») и статусе проекта.
- **Целостность**: Каждое действие в системе логируется с привязкой к конкретному идентификатору заявки (`application_id`).

## Структура проекта

- `bot.js` — точка входа, инициализация и обработка экшенов.
- `src/db.js` — уровень работы с базой данных (PostgreSQL).
- `src/menus.js` — тексты всех сообщений и структуры клавиатур.
- `src/*-flow.js` — логика конкретных процессов (подача заявки, администрирование, статистика).
